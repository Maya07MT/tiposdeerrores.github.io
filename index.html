Introduction

Like all things created by humans, computer programs and the data they process can contain errors. Errors are conditions that are not part of the normal operation of a program and can be syntax, semantic or execution.

Syntax errors are problems with code written by the programmer
Semantic errors are problems with algorithms that prevent the expected result from returning
Execution errors are conditions that affect the normal operation of the program and can be caused by multiple circumstances

Our research focuses on the different types of errors in programming as well as the technique of cause-effect graphs

Syntax Error

A syntax error occurs when the program does not meet the language rules. When this error occurs, it means that the program is misspelled. The error name is SyntaxError.

Syntax errors always occur before the program is executed. That is, a poorly written program fails to execute any instruction. Thus, the syntax error is no exception.

Warnings

Warnings are errors, but not serious enough to prevent object code generation.

However, it is important to correct these errors most of the time because before a warning the compiler has to make decisions

and these do not have to coincide with what we intend to do, they are already based on the directives that compiler creators decided during compiler creation.

Execution Error

These errors are more difficult to detect and correct (as it is the logic as such of our application). There are auxiliary tools to search for these errors,
they are called debuggers (debuggers). Even after obtaining an executable file, errors may occur during code execution.

Even after obtaining an executable file, errors may occur during code execution.

Design Error

The most difficult mistakes to correct and prevent. If we have been wrong in designing our algorithm,
there will be no program that can help us correct them,

because it is impossible for a program to determine what we are trying to achieve or a program that makes any applications for us.

Linking Error

The linker program may also encounter errors. They usually refer to functions that are not defined in any of the object files or libraries.

We may have forgotten to include a library or an object file, or we may have forgotten to define a function or variable, or we may have done it wrong.
Broken internal links: This may be due to an incorrect formatted URL.
Broken external links: they point to an external site that no longer exists or has been moved and has not implemented any redirection.
Too many links on the page: The more links there are, the lower the SEO value

Integrity Error

People who log in can make mistakes, leading to variations between the original data and the data stored in a system.
Similarly, people can make mistakes during the transfer or electronic copy of data, making disparity between different versions or references to a file.
If the system applies integrity to the data, it will prevent the user from making these errors.

Maintaining data integrity means ensuring that data remains intact and unchanged throughout its entire life cycle.

Technique of Cause-Effect Graphs

It is a technique that allows the tester to validate complex sets of actions and conditions. Cause-effect graphs represent the logical relationship between different causes and possible effects.
For this we list the causes (user inputs or actions) and the effects
(expected system outputs or actions), and then join indicating relationships between them.

It is a test case technique that provides a concise representation of the logical conditions and their corresponding actions.

The technique follows four steps:

Causes (input conditions) and effects (actions) are listed for a module, assigning an identifier to each of them.
A Cause-Effect graph is developed.
The graph becomes a decision table.
The decision table rules are converted to test cases.
